Week 16 Part 4 - 練習與總結

學習目標：實作練習、專案延伸、課程總結

本週練習題

基礎練習（必做）

  1. 新增科目支援
    擴充 Student 類別，新增「物理」和「化學」兩個科目，修改相關計算方法。
    • 修改 __init__ 方法增加兩個參數
    • 更新 get_total_score() 和 get_average_score()
    • 更新 to_dict() 和 from_dict()
    • 測試所有功能是否正常運作
  2. 成績分級統計
    在 StudentManager 中新增方法 get_grade_distribution()，統計各等級（A/B/C/D/F）的學生人數和百分比。
    • 返回包含各等級統計的字典
    • 計算每個等級的人數
    • 計算每個等級的百分比
    • 在主程式中新增選單項目顯示此統計
  3. 輸入驗證強化
    加強 input_student_data() 的驗證功能：
    • 學號格式檢查（例如：必須是 S 開頭加上三位數字）
    • 姓名長度限制（2-10 個字元）
    • 不允許輸入特殊符號
    • 提供更詳細的錯誤訊息

進階練習

  1. 單科成績排名
    實作單科成績排名功能：
    • 在 StudentManager 新增 get_subject_ranking(subject) 方法
    • 可指定科目（chinese/english/math）
    • 返回該科目的排名列表
    • 在主程式新增選單選項
  2. 批次匯入功能
    實作從 CSV 檔案批次匯入學生資料：
    • 新增 import_from_csv(filename) 方法
    • 讀取 CSV 檔案並建立學生物件
    • 處理格式錯誤的資料
    • 顯示匯入結果（成功/失敗筆數）
  3. 成績趨勢分析
    為每個學生記錄歷史成績，實作趨勢分析：
    • 修改資料結構支援多次成績記錄
    • 計算成績變化趨勢（進步/退步）
    • 顯示歷史成績圖表（使用文字圖形）
    • 計算最高/最低/平均成績

挑戰練習

  1. 圖形化介面（GUI）
    使用 tkinter 或 PyQt5 建立圖形化介面：
    • 設計主視窗和各功能視窗
    • 使用表格顯示學生列表
    • 提供表單輸入介面
    • 加入圖表顯示統計資料
  2. Web 介面
    使用 Flask 或 Django 建立 Web 版本：
    • 設計 RESTful API
    • 建立前端網頁介面
    • 實作使用者登入功能
    • 支援多使用者同時操作
  3. 資料庫整合
    將資料儲存改為使用資料庫（SQLite/MySQL）：
    • 設計資料庫結構
    • 使用 SQL 或 ORM（SQLAlchemy）
    • 實作資料庫 CRUD 操作
    • 加入資料備份與還原功能
練習提示：
  • 從基礎練習開始，循序漸進
  • 每完成一個功能就進行測試
  • 保持程式碼的可讀性和結構性
  • 適當添加註解說明
  • 處理各種可能的錯誤情況
  • 參考現有程式碼的設計模式

技術升級方向

1. 圖形化介面 (GUI)

  • PyQt5/PySide6: 功能強大，介面美觀
  • Kivy: 支援跨平台和觸控

2. Web 應用

  • Flask: 輕量級，適合小型專案
  • Django: 功能完整，適合大型專案
  • FastAPI: 現代化，效能優異

3. 資料庫

  • SQLite: 輕量級，無需安裝
  • MySQL/PostgreSQL: 功能完整的關聯式資料庫

4. 資料視覺化

  • matplotlib: 基本圖表繪製
  • seaborn: 統計視覺化
  • plotly: 互動式圖表

開發最佳實務

1. 程式碼組織

模組化設計原則：
  • 一個檔案只負責一個主要功能
  • 類別和函數保持單一職責
  • 避免過度耦合
  • 使用清晰的命名規則

2. 錯誤處理

異常處理策略：
  • 預期可能發生的錯誤要明確處理
  • 提供有意義的錯誤訊息
  • 避免吞沒例外（silent failure）
  • 記錄錯誤日誌供除錯使用

3. 資料驗證

輸入驗證要點：
  • 永遠不要信任使用者輸入
  • 檢查資料型態和格式
  • 驗證數值範圍
  • 防止 SQL 注入等安全問題

4. 測試

測試策略：
  • 單元測試：測試個別函數和方法
  • 整合測試：測試模組間的協作
  • 使用 pytest 或 unittest 框架
  • 追求高程式碼覆蓋率

5. 文件

文件撰寫：
  • 使用 docstring 記錄函數用途
  • 撰寫 README 說明專案
  • 記錄 API 介面規格
  • 提供使用範例

6. 版本控制

使用 Git：
  • 每個功能使用獨立分支開發
  • commit 訊息要清楚說明改動
  • 定期推送到遠端倉庫
  • 使用 .gitignore 排除不必要的檔案
